Usage of c45

c45 is used to build decision trees for the specified data set. It prunes the decision tree and print it in the specified manner.

Usage: c45 <dataSetName> [-output plain|html -repeat times]

  1. Option -output or -o specifies the output manner.
    • plain output is adopted by default.
    • html files produced by html output must be put together with the dTree directory unzipped from ../lib/dTree.zip.
  2. Option -repeat or -r specifies the times of repetition to accurately measure the tree-building time.

For example: c45 ../dataset/UCI/golf

Usage of cv

cv is used to cross validate the classification performance of the c45 decision tree.

Usage: cv <dataSetName> [fold]

  1. Option fold specifies the number of fold of the cross validation (10 by default).

For example: cv ../dataset/UCI/golf

Usage of create

create is used to create executable files in the ../work directory for the specified data set stored in ../dataset/UCI/ directory.

The functions of the created files are as follows

  1. "plainview" produces plain view of the decision tree.
  2. "htmlview" produces html view of the decision tree.
  3. "xmlview" produces xml view of the decision tree.
  4. "hibernate" performs object persistence from decision tree to xml file.
  5. "spring" restores the object decision tree from xml file and output it in html view.
  6. "clean" removes all the produced files.

The above created executable files provide an easier way to build decision trees, print decision trees in different ways, transforms decision tree to xml file and restore it from xml file.

Usage: create <dataSetName>

For example: create golf
