The format of the .names file

The format of the .names file is as follows:

For a discrete attribute, its attribute name should be first provided, then a colon followed with its all nominal values.
For example: outlook: sunny, overcast, rain

For a continuous attribute, its attribute name should also be first provided, then a colon followed with a string of continuous.
For example: temperature: continuous

To specify the target (or class) attribute, you can write the target attribute: xxxx at the top of the file.
For example: the target attribute: outlook
If no target attribute is specified, the attribute with the name of class is taken as the target attribute by default.

A complete example: golf.names

the target attribute: class
outlook: sunny, overcast, rain
temperature: continuous
humidity: continuous
windy: true, false
class: Play, Don't Play

The format of the .data file

The format of the .data file is the same as the UCI format, i.e. attribute values separated with commas.
For example: sunny, 85, 85, false, Don't Play

A complete example: golf.data

sunny, 85, 85, false, Don't Play
sunny, 80, 90, true, Don't Play
overcast, 83, 78, false, Play
rain, 70, 96, false, Play
rain, 68, 80, false, Play

The format of the .test file

The format of the .test file is the same as the .data file.
